# Claudio 音频可视化全面优化 — Bug 修复 + 律动增强

## 第一部分：修复 3 个致命 Bug

### BUG-1: SpectrumBars `Cannot read properties of null (reading 'connect')`
**文件**: `apps/web/src/components/SpectrumBars.tsx`
**错误**: `sourceNode.connect(analyser)` 第 27 行，因为 `audioPlayer.getSourceNode()` 返回 null
**原因**: AudioPlayer 的 `ensureGainNode()` 在 try-catch 里失败后 sourceNode 仍为 null
**修复方案**:
1. AudioPlayer.ts `ensureGainNode()`: catch 中重置 `this.audioCtx = null; this.sourceNode = null; this.gainNode = null;`
2. SpectrumBars: 在 `setupAudio()` 中检查 sourceNode 是否为 null，如果为 null 则直接 return（不 crash），等下一次 active 变化再试。或者改为延迟初始化 — 在 draw 循环里检测 analyserRef.current 是否为 null，如果为 null 再调用 setupAudio()

### BUG-2: AudioVisualizer 自己创建 AudioContext 导致冲突
**文件**: `apps/web/src/components/AudioVisualizer.tsx` 第 37-52 行
**当前代码**:
```ts
const audioCtx = new ACtor();  // 第 41 行
const source = audioCtx.createMediaElementSource(audioEl);  // 第 45 行
```
**问题**: 同一个 audio element 上调用两次 createMediaElementSource 会抛 InvalidStateError
**修复**: 删除整个 AudioVisualizer 的 AudioContext/sourceNode/analyser 创建逻辑。改为复用 audioPlayer 的共享资源：
```ts
const setupAudio = useCallback(() => {
  if (analyserRef.current) return;
  try {
    const audioCtx = audioPlayer.getAudioContext();
    const sourceNode = audioPlayer.getSourceNode();
    if (!audioCtx || !sourceNode) return; // null guard
    const analyser = audioCtx.createAnalyser();
    analyser.fftSize = 256;
    analyser.smoothingTimeConstant = 0.8;
    sourceNode.connect(analyser);
    analyserRef.current = analyser;
    freqDataRef.current = new Uint8Array(analyser.frequencyBinCount);
  } catch (err) {
    console.warn("[AudioVisualizer] Could not set up audio analysis:", err);
  }
}, []);
```
同时删除 `audioCtxRef` 和 `sourceRef` 这两个 ref，不再需要了。

### BUG-3: `Maximum update depth exceeded` — 无限渲染循环
**调用链**:
1. `PlayerPage.tsx:181` — `<AudioVisualizer onFrequencyData={(b, m) => { setBass(b); setMid(m); }} />`
2. 每次 PlayerPage 渲染，箭头函数是新引用
3. AudioVisualizer 的 draw() 每帧调用 `onFrequencyData(bass, mid)` → setBass/setMid → PlayerPage 重渲染 → 新回调引用
4. AudioVisualizer `useEffect(..., [mode, onFrequencyData])` 检测到 onFrequencyData 变化 → 重跑 effect → 新 draw loop → 死循环

**修复（两处）**:

**PlayerPage.tsx** — 把回调用 useCallback 包裹：
```tsx
// 在 PlayerPage 组件内，useEffect 之前添加：
const handleFrequencyData = useCallback((b: number, m: number) => {
  setBass(b);
  setMid(m);
}, []);

// 第 181 行改为：
<AudioVisualizer mode={visualMode} onFrequencyData={handleFrequencyData} />
```

**AudioVisualizer.tsx** — 把 onFrequencyData 存入 ref，useEffect 依赖不包含它：
```tsx
const onFreqRef = useRef(onFrequencyData);
onFreqRef.current = onFrequencyData;

// useEffect 依赖改为只包含 mode：
useEffect(() => {
  // ... draw loop 里用 onFreqRef.current(bass, mid) 而不是 onFrequencyData(bass, mid)
}, [mode]); // ← 不再包含 onFrequencyData
```

同理，AudioVisualizer 的 `setupAudio` useEffect 第 54-62 行不需要改（它不依赖 onFrequencyData）。

## 第二部分：增强所有视觉组件的音乐律动

### 2.1 AudioVisualizer.tsx — 更强的 bass/mid 响应
**当前问题**: 视觉效果太微弱，bass 和 mid 对形状影响太小

修改 ATTACK 和 RELEASE 常量：
- `ATTACK = 0.4` → `0.6` （响应更快）
- `RELEASE = 0.15` → `0.3` （恢复更快）

修改 draw() 里的乘数：
- `speedMul = 1 + bass * 0.6` → `1 + bass * 2.5`
- `scaleMul = 1 + bass * 0.4` → `1 + bass * 1.8`

每个模式增强：
- **Glob**: 半径基础值增大，加入 mid 驱动的细节变形
  ```ts
  const r = (100 + Math.sin(time * 2.5 * speedMul + i * 0.3) * 50 + Math.cos(time * 1.8 * speedMul + i * 0.5) * 35) * scaleMul;
  ```
- **Flower**: 花瓣位移和大小随 bass 更剧烈
  ```ts
  const r = (80 + Math.sin(time * 2.5 * speedMul + p) * 35) * scaleMul;
  // 位移也乘 scaleMul
  cx + Math.cos(angle) * 45 * scaleMul
  ```
- **Arcs**: 弧线间距随 bass 脉动
  ```ts
  const r = (50 + i * (8 + bass * 6)) * scaleMul;
  ```
- **Circles**: 圈圈脉冲感更强
  ```ts
  const r = (40 + i * 30 + Math.sin(time * 3 * speedMul + i) * 20) * scaleMul;
  ```
- **Wave**: 振幅加倍
  ```ts
  const y = cy + Math.sin(x * 0.02 + time * 4 * speedMul) * (50 + bass * 40) + Math.sin(x * 0.01 + time * 2.5 * speedMul) * (35 + mid * 25);
  ```
- **Shine**: 光线长度变化更剧烈
  ```ts
  const len = (50 + Math.sin(time * 4 * speedMul + i * 0.5) * 45) * scaleMul;
  ```

透明度也提高：
- `opBase = isPlaying ? 0.45 : 0.15` （当前是 0.3 / 0.15）

### 2.2 FluidBlobs.tsx — 更活跃的流体
**当前**: bass 乘数 `b * 0.4` 和 `b * 0.35` 太保守

增强：
```ts
// primary blob
const drift = 5 + b * 8;           // 当前 2 + b * 3

```

### 2.3 SpectrumBars.tsx — 更有冲击力的频谱
增强 draw() 函数：
- 条形高度乘以 1.3 倍：`const barHeight = bars[i] * h * 1.3;`
- 渐变更亮：
  ```ts
  grad.addColorStop(0, "rgba(255,255,255,0)");
  ```

### 2.4 WaveformBar.tsx — 跟着音乐跳动
**当前问题**: 高度是 useMemo 生成的固定值，不随音乐变化

改造方案：
- 接受 bass 和 mid props（从 PlayerPage 传入）
- 在 draw 中让每个 bar 的高度随 bass 脉动：
  ```tsx
  style={{ height: `${height * (1 + bass * 0.8)}px` }}
  ```
- 或者用 CSS transition 让变化更平滑
- 需要从 PlayerPage 传 bass 进来：`<WaveformBar barCount={60} bass={bass} />`

### 2.5 BorderGlow.tsx — 跟随 bass 脉冲发光
**当前**: 固定的 amp=5 和固定 alpha

改造方案：
- 通过 CSS 变量或者直接读取 playerStore 来获取 bass 值
- 在 draw() 里：
  ```ts
  // 读取 bass 值（从 CSS 变量或全局变量）
  const bassVal = parseFloat(document.documentElement.style.getPropertyValue('--audio-bass') || '0');
  const amp = 5 + bassVal * 15;  // bass 强时振幅更大
  ```
- alpha 也随 bass 增强：
  ```ts
  const layers = [
    { w: 40 + bassVal * 20, blur: 30, alpha: 0.03 + bassVal * 0.05 },
    { w: 25 + bassVal * 15, blur: 15, alpha: 0.06 + bassVal * 0.08 },
    { w: 12 + bassVal * 8, blur: 6, alpha: 0.1 + bassVal * 0.1 },
    { w: 6 + bassVal * 4, blur: 2, alpha: 0.15 + bassVal * 0.15 },
  ];
  ```

为了让 BorderGlow 能获取 bass，需要在 PlayerPage 里把 bass 写入 CSS 变量：
```tsx
// 在 PlayerPage 的 render 或 useEffect 里：
useEffect(() => {
  document.documentElement.style.setProperty('--audio-bass', String(bass));
}, [bass]);
```

### 2.6 PlayerPage.tsx cover glow 增强

增强为：

```ts
const shadow = 40 + bass * 80;
```
