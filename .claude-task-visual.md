## 2. 修复 AudioVisualizer VISUAL MODE 可见

文件: `apps/web/src/components/AudioVisualizer.tsx` + `apps/web/src/styles/global.css`

**核心问题**: CSS 里 `.viz-canvas` 设了 `opacity: 0.15; filter: blur(2px)`，导致 6 种模式切换根本看不出来。

修改方案:
- CSS: `.viz-canvas` opacity 改为 `0.5`，移除 blur filter
- AudioVisualizer.tsx: 把所有硬编码的 `mint` 颜色改为使用 CSS 变量 `--color-primary`
- 具体改法: 把 `const mint = "rgba(94,232,197,";` 改为从 CSS 变量读取:

```tsx
// 在 draw 循环内读取最新颜色
const style = getComputedStyle(document.documentElement);
const primaryColor = style.getPropertyValue('--color-primary').trim() || '#5ee8c5';
// 解析 rgb(r,g,b) 或 hex 为 rgba 字符串
function toRgba(color: string, alpha: number): string {
  if (color.startsWith('rgb(')) {
    return color.replace('rgb(', 'rgba(').replace(')', `,${alpha})`);
  }
  // hex
  const m = color.match(/^#([0-9a-f]{6})$/i);
  if (m) {
    const r = parseInt(m[1], 16) >> 16;
    const g = (parseInt(m[1], 16) >> 8) & 0xff;
    const b = parseInt(m[1], 16) & 0xff;
    return `rgba(${r},${g},${b},${alpha})`;
  }
  return `rgba(94,232,197,${alpha})`;
}
```

然后所有 `${mint}${opBase})` 替换为 `toRgba(primaryColor, opBase)`

## 3. 修复 BorderGlow 动态颜色

文件: `apps/web/src/components/BorderGlow.tsx`

当前硬编码: `ctx.strokeStyle = 'rgba(170, 130, 255, ...)'`

改为从 CSS 变量 `--color-primary` 读取颜色:

```tsx
const style = getComputedStyle(document.documentElement);
const pc = style.getPropertyValue('--color-primary').trim() || '#5ee8c5';
// 解析为 r,g,b
function parseColor(c: string): [number, number, number] {
  if (c.startsWith('rgb(')) {
    const m = c.match(/(\d+)/g);
    return m ? [+m[0], +m[1], +m[2]] : [94, 232, 197];
  }
  const m = c.match(/^#([0-9a-f]{6})$/i);
  if (m) return [parseInt(m[1], 16) >> 16, (parseInt(m[1], 16) >> 8) & 0xff, parseInt(m[1], 16) & 0xff];
  return [94, 232, 197];
}
const [cr, cg, cb] = parseColor(pc);
// 然后
ctx.strokeStyle = `rgba(${cr},${cg},${cb},${l.alpha})`;
```

## 4. 新增底部频谱条组件

参考 kabaBZ 的 `visual.js` 第 163-241 行的频谱实现。

新建文件: `apps/web/src/components/SpectrumBars.tsx`

设计:
- 固定在页面底部，宽度 440px 或 100%（取小值），高 80px
- 64 条频谱柱
- 频率映射: `Math.pow(i / BAR_COUNT, 1.6)` 做非线性映射
- idle 时呼吸漂动: `0.06 + 0.04 * Math.sin(t * 0.002 + i * 0.3)`
- 平滑: `specBars[i] += (target - specBars[i]) * (target > specBars[i] ? 0.35 : 0.12)`
- 颜色: 底部透明 → 顶部半透明白色渐变
- 用 AudioVisualizer 已有的 onFrequencyData 回传递 bass/mid 到父组件，但频谱组件需要原始 freqData

**更好的方案**: 让 AudioVisualizer 通过一个新的 onRawFreqData 回调暴露原始频谱数据，或让 SpectrumBars 自己连接 AudioPlayer 的 Web Audio。

最简方案: SpectrumBars 内部自行连接 audioPlayer，不依赖 AudioVisualizer。参考 AudioVisualizer 的 setupAudio 模式。

Props: `{ active: boolean }` — active=true 时显示实时频谱，false 时显示 idle 呼吸

集成到 PlayerPage: 在 player-card 下方添加 `<SpectrumBars active={isPlaying} />`

CSS:

## 5. 封面律动光效

文件: `apps/web/src/pages/PlayerPage.tsx`

在 player-upper 里已有封面区域，但没有 cover-glow 元素。

参考 kabaBZ `visual.js` 第 291-298 行:
- `#coverGlow` 在封面区域，bass 驱动 `box-shadow` 脉动
- idle 时 `opacity: 0.3 + 0.05 * Math.sin(t * 0.001)`
- 播放时 `box-shadow: 0 0 ${30 + bass * 40}px ${8 + bass * 12}px var(--color-primary)`

需要:
1. 在 PlayerPage 中加一个 `coverGlowRef` 状态
2. 用 `useEffect` + `requestAnimationFrame` 驱动 glow 动画
3. 在播放器 upper 区域的适当位置添加 `.cover-glow` div
4. CSS `.cover-glow` 已在 global.css 中定义
