# Task 2: 聊天系统升级

项目路径: /projects/Claudio
包名: @ai-radio/web (apps/web), @ai-radio/server (apps/server)

## 任务总览

升级聊天系统：消息分流引擎、结构化回复、歌曲卡片组件、agent.md 配置。

## 1. 创建 agent.md 配置文件

新建: `config/agent.md`

内容参考 kabaBZ 设计文档描述的 Claudio 人设:
```markdown
# Claudio — AI DJ

## 身份
你是 Claudio，一个沉浸式 AI 电台的 DJ。你的工作是为用户挑选音乐、分享见解、与用户互动。

## 说话风格
- 使用 DJ 口吻，轻松有趣，偶尔中英混搭
- 分享歌曲故事、创作背景、冷知识
- 回复简洁有力，避免冗长

## 行为准则
- 用户说"下一首"等指令时直接执行，不需要废话
- 推荐歌曲时给出简短推荐理由
- 当用户情绪低落时，推荐舒缓的音乐
- 当用户需要专注时，推荐轻音乐或 lo-fi
```

## 2. 后端: /api/dispatch 消息分流

新建: `apps/server/src/routes/dispatch.ts`

分流逻辑:
```
用户输入 → 前端 POST /api/dispatch
  ↓
意图识别（关键词匹配）:
  - 简单指令: "下一首|上一首|暂停|播放|音量|随机|循环" → 返回 `{type: "command", action: "next|prev|pause|play|..."}`
  - 音乐搜索: "搜索xxx|播放xxx|找xxx" → 返回 `{type: "search", query: "xxx"}`
  - 自然语言: 其他 → 转发给 Claude API
```

在 `apps/server/src/index.ts` 中注册:
```ts
import { dispatchRoutes } from "./routes/dispatch.js";
await app.register(dispatchRoutes);
```

API 设计:
- POST `/api/dispatch` body: `{ message: string }`
- 返回:
  - 指令类: `{ type: "command", action: string, message?: string }`
  - 搜索类: `{ type: "search", query: string, results: [...] }`
  - 聊天类: Claude 流式 SSE 响应（复用现有 /api/chat 逻辑）

## 3. 后端: 结构化 Claude 回复

修改: `apps/server/src/services/claude.service.ts`

Claude 回复必须是 JSON:
```json
{
  "say": "回复给用户的消息文本",
  "reason": "推荐理由",
  "play": [{"id": "歌曲ID", "name": "歌曲名", "artist": "艺术家", "album": "专辑", "cover": "封面URL"}],
  "segue": "要用 TTS 朗读的语音内容"
}
```

修改 system prompt 拼装: 加载 agent.md + taste.md + routines.md + moodrules.md

解析 Claude 回复: 尝试 JSON.parse，如果失败则当作纯文本 `{say: rawText}`

## 4. 前端: 歌曲卡片组件

新建: `apps/web/src/components/SongCard.tsx`

设计:
```
┌─────────────────────────────┐
│ [封面] 歌曲名                │
│        艺术家    ▶ + │
└─────────────────────────────┘
```

- `▶` 按钮: 调用 `usePlayerStore.getState().playItem()` 播放
- `+` 按钮: 添加到当前队列
- 无直链处理: 灰色遮罩 + 禁用按钮 + "暂无音源" 提示

Props:
```ts
interface SongCardProps {
  songId: string;
  title: string;
  artist: string;
  coverUrl?: string;
  album?: string;
  onPlay?: () => void;
  onAdd?: () => void;
  disabled?: boolean; // 无直链时置灰
}
```

## 5. 前端: ChatMessage 解析结构化回复

修改: `apps/web/src/components/ChatArea.tsx`

每条 assistant 消息解析:
1. 尝试 JSON.parse
2. 如果包含 `say` → 渲染文字
3. 如果包含 `play` → 渲染 SongCard 列表
4. 如果包含 `segue` → 渲染语音波形条（可点击播放）
5. 如果 parse 失败 → 当纯文本渲染

展示顺序（在气泡内从上到下）:
1. 文字: say + reason
2. 语音: segue 波形条
3. 歌曲: play 的 SongCard 列表

## 6. 前端: IntentInput 使用 dispatch

修改: `apps/web/src/components/IntentInput.tsx`

发送消息时:
1. POST `/api/dispatch` `{ message }`
2. 如果返回 `type: "command"` → 执行指令 + toast
3. 如果返回 `type: "search"` → 展示搜索结果
4. 如果是 SSE 流 → 解析结构化 JSON 渲染
