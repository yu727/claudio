# AI 音乐电台产品文档

## 1. 产品概述

本产品是一个运行在本地或私有环境中的 AI 音乐电台。用户导入或授权自己的网易云歌单后，系统结合用户品味、日程、天气、时间、历史播放与即时指令，由 Claude 生成播放计划和主持串词，并通过语音合成生成类似电台 DJ 的播报。前端以 PWA 播放器形式提供播放、切歌、个人资料、设置和状态展示。

一句话定位：把用户多年沉淀的歌单蒸馏成一个会看场景、会说话、会选歌的个人 AI 电台。

## 2. 目标用户

- 长期使用网易云音乐、歌单积累较多的用户。
- 想要“被理解”的音乐播放体验，而不是纯随机或平台推荐。
- 喜欢 AI 创作、vibe coding、本地自动化和个人数字助理的用户。
- 希望在工作、通勤、运动、夜晚等场景中获得自动配乐和语音陪伴的人。

## 3. 核心价值

- 个性化：基于用户歌单、口味规则、情绪规则和播放历史生成推荐。
- 场景感：结合天气、日历、时间段、状态记忆，动态调整音乐和播报。
- 电台感：不仅播放歌曲，还生成 DJ 串词、天气提醒、日程提示、音乐介绍。
- 可控性：用户可以通过自然语言指令、设置页和规则文件调整电台风格。
- 私有化：核心服务运行在 localhost 或自部署服务器，用户数据优先本地保存。

## 4. 使用场景

### 4.1 日常自动播放

用户打开 PWA 播放器，系统自动根据当前时间、天气和最近播放记录生成一段播放计划。播放器开始播放歌曲，并在歌曲间插入 AI DJ 语音。

### 4.2 早晨启动

每天早上 07:00，系统读取日历、天气和用户近期偏好，生成早间播放清单和简短播报。例如提醒今日天气、行程密度，并选择轻快或稳定的音乐。

### 4.3 即时点播

用户在播放器输入“来点适合写代码的歌”或“今晚想听 2012 年附近的老歌”，系统将请求交给 Claude 解析，转换为搜索、推荐和播放计划。

### 4.4 歌单蒸馏

系统导入用户多年歌单，分析常听歌手、风格、年代、语言、情绪标签，形成 taste.md、routines.md、mood-rules.md 等用户画像文件。

### 4.5 家庭音响播放

本地服务通过 UPnP 将音乐推送到家庭音响或局域网播放设备，PWA 作为控制器。

## 5. 产品范围

### 5.1 MVP 必做

- PWA 播放器：首页播放器、播放/暂停、上一首/下一首、播放队列、当前歌曲信息。
- 网易云音乐接入：搜索歌曲、获取歌曲 URL、获取歌词、基础推荐。
- Claude 编排：根据上下文生成播放计划、DJ 串词和工具调用意图。
- Fish Audio TTS：将串词合成为音频，并插入播放队列。
- 本地 Node.js 服务：统一管理 API、状态、缓存、任务调度。
- 状态记忆：保存 messages、plays、plans、当前队列、最近偏好。
- 设置页：配置 API Key、网易云登录信息、TTS 声音、Claude 模型、播报频率。
- 每日计划：按固定时间生成今日播放策略。

### 5.2 V1 增强

- 日历接入：读取飞书日历，影响早间播报和播放强度。
- 天气接入：读取 OpenWeather，生成天气播报和情绪调节。
- UPnP 投放：发现设备、选择设备、推送播放。
- Profile 页：展示口味画像、常听歌手、最近主题、播放统计。
- 自然语言控制：用户输入一句话即可调整氛围、风格、音量、模式。
- 歌单分析任务：定期重新分析歌单并更新用户语料。

### 5.3 暂不做

- 多用户账号体系。
- 公开社交、电台分享、评论互动。
- 原生移动 App。
- 大规模云端推荐模型训练。
- 付费系统。

## 6. 信息架构

PWA 包含三个主页面：

- Player：播放器主界面。
- Profile：用户画像、口味、播放统计。
- Settings：API Key、登录、声音、调度、缓存、设备设置。

推荐底部导航：

- 播放
- 队列
- 画像
- 设置

## 7. 播放器界面需求

### 7.1 视觉风格

- 深色背景，偏个人电台和音乐播放器气质。
- 主体区域突出封面、歌名、歌手和当前 AI DJ 状态。
- 操作按钮使用清晰图标：播放、暂停、上一首、下一首、收藏、切换模式。
- 信息密度适中，不做营销式落地页。

### 7.2 页面元素

- 当前封面图。
- 歌曲名、歌手、专辑。
- AI DJ 播报状态：生成中、播报中、已跳过。
- 进度条和时间。
- 播放控制按钮。
- 播放模式：顺序、随机、场景电台。
- 当前场景标签：如“早晨”“写代码”“下雨”“通勤”。
- 队列入口。
- 文本指令输入框。

### 7.3 关键交互

- 点击播放：播放当前队列第一条音频。
- 点击下一首：跳过当前音频，并记录 skip 行为。
- 输入自然语言：提交到 `/api/intent`，由后端生成新的播放计划或操作。
- 长按歌曲：展示收藏、拉黑、更多相似、查看歌词。
- 点击 AI 播报：可跳过、重播或降低播报频率。

## 8. Profile 页面需求

展示用户画像：

- 常听歌手 Top 20。
- 常听年代分布。
- 常听语言分布。
- 情绪偏好：安静、燃、怀旧、电子、民谣等。
- 最近 7 天播放主题。
- 歌单导入状态。
- AI 总结的“你的音乐人格”。

用户可以编辑：

- 喜欢的电台人格。
- 不喜欢的播报方式。
- 禁播歌手/歌曲。
- 不同时间段的音乐偏好。

## 9. Settings 页面需求

设置项：

- Claude API Key / Base URL / 模型。
- 网易云音乐登录 Cookie 或账号登录方式。
- Fish Audio API Key / Voice ID。
- OpenWeather API Key / 城市。
- 飞书 API 配置。
- UPnP 设备选择。
- 播报频率：低、中、高、只早晚。
- TTS 缓存上限。
- 歌曲缓存上限。
- 本地服务地址。

## 10. AI DJ 行为规则

AI DJ 应该：

- 说短句，避免过度打扰。
- 结合天气、时间、日程和播放历史。
- 能解释为什么接下来播放某首歌。
- 避免重复相同句式。
- 避免虚构用户没有提供的私人信息。
- 当工具调用失败时，用自然方式降级，不暴露技术错误。

AI DJ 不应该：

- 每首歌都长篇介绍。
- 频繁打断音乐。
- 说“我是一个 AI 模型”之类破坏沉浸感的话。
- 生成版权歌词长段引用。
- 在没有授权的情况下上传用户歌单或播放历史。

## 11. 播放计划规则

播放计划由若干 item 组成：

- `song`：真实歌曲。
- `tts`：AI DJ 播报音频。
- `silence`：短暂停顿，可选。

示例：

```json
{
  "scene": "morning",
  "items": [
    { "type": "tts", "text": "早上好，今天有点阴，先从轻一点的歌开始。" },
    { "type": "song", "songId": "123", "reason": "符合早晨和轻快偏好" },
    { "type": "song", "songId": "456", "reason": "延续同一年代的怀旧感" }
  ]
}
```

## 12. 成功指标

- 用户首次导入歌单后 3 分钟内能开始播放。
- AI DJ 首次播报生成时间小于 10 秒。
- 歌曲切换响应小于 1 秒。
- 70% 以上播放计划不需要用户手动跳过前三首。
- 用户一周内至少打开 3 次。
- TTS 缓存命中率随使用增长，重复播报不重复合成。

## 13. 风险与限制

- 网易云音乐非官方接口可能不稳定，需要降级策略。
- 部分歌曲 URL 可能不可播放，需要自动换源或跳过。
- TTS 成本需要缓存控制。
- Claude 工具调用需要严格限制，避免任意执行危险命令。
- 日历和歌单属于隐私数据，默认本地保存，清晰提示用户授权范围。
