## 1. SQLite 表结构补充

修改: `apps/server/src/db/schema.ts` 或类似文件

需要检查现有 schema，补充以下表（如果不存在）:

```sql
-- 播放历史
CREATE TABLE IF NOT EXISTS play_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  song_id TEXT,
  song_name TEXT,
  artist TEXT,
  album TEXT,
  cover_url TEXT,
  played_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 聊天消息
CREATE TABLE IF NOT EXISTS chat_messages (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  role TEXT NOT NULL CHECK(role IN ('user', 'assistant')),
  content TEXT NOT NULL,
  song_cards TEXT,  -- JSON
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 偏好/配置
CREATE TABLE IF NOT EXISTS preferences (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL
);

-- 播放状态（续播用）
CREATE TABLE IF NOT EXISTS playback_state (
  id INTEGER PRIMARY KEY CHECK(id = 1),
  current_song_id TEXT,
  current_song_name TEXT,
  current_song_artist TEXT,
  current_song_album TEXT,
  current_song_cover TEXT,
  progress_seconds REAL DEFAULT 0,
  queue_song_ids TEXT,  -- JSON
  queue_index INTEGER DEFAULT 0,
  play_mode TEXT DEFAULT 'off',
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 6. 定时任务调度器

新建: `apps/server/src/services/scheduler.service.ts`

使用 node-cron 包（需要添加到 package.json dependencies）。

### 安装依赖
```bash
cd /projects/Claudio/apps/server && pnpm add node-cron && pnpm add -D @types/node-cron
```

### 调度任务

**每日歌单推荐（07:00）**
- 读取 taste.md + 最近播放历史 + moodrules.md
- 调用 Claude API 生成推荐歌单
- 通过 NCM API 搜索匹配歌曲
- 存入 playlists 表（type: daily）
- 存入 preferences 表（key: daily_playlist）

**每小时情绪检查**
- 读取 moodrules.md + 当前时间 + 最近聊天记录
- 调用 Claude API 判断情绪标签
- 存入 preferences 表（key: current_mood）

**品味画像生成（每日 06:00）**
- 读取收藏列表 + 播放历史 + taste.md + moodrules.md
- 调用 Claude 以 DJ 口吻生成品味总结
- 存入 preferences 表（key: taste_profile）

### 调度器 API

新建: `apps/server/src/routes/scheduler.ts`

```
GET  /api/scheduler/status          → 各任务运行状态
GET  /api/scheduler/daily-playlist  → 今日推荐歌单
GET  /api/scheduler/mood            → 当前情绪
POST /api/scheduler/trigger/:task   → 手动触发任务（调试用）
```
