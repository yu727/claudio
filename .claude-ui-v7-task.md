# Claudio UI 重构任务

## 任务概览
三个页面大改造：歌单、品味、设置。

## 1. 歌单页面 — 显示收藏歌曲

### 当前状态
`PlaylistPage.tsx` 有两个 tab：本地歌单 + 网易云歌单。没有显示用户收藏的歌曲。

### 需求
- 新增第三个 tab「❤️ 收藏」(Favorites)，作为**默认选中**的 tab
- 显示用户收藏的所有歌曲（来自 SQLite favorites 表）
- 每首歌显示封面、标题、艺术家、时长
- 点击歌曲直接播放
- 有「播放全部」按钮
- 有「取消收藏」按钮（每首歌右边，点击后从列表移除）

### 后端
已有 API：
- `GET /api/favorites` → `{ favorites: FavoriteItem[] }` （在 plays.repo.ts 的 getFavorites()）
- `DELETE /api/favorites/:songId` → 取消收藏
- FavoriteItem: `{ songId, title, artist, coverUrl, createdAt }`

### 前端改动
- `PlaylistPage.tsx`: 新增 tab "favorites"，默认选中
- 收藏列表组件，复用现有 pl-track 样式
- API client 已有 `getFavorites()` 和 `removeFavorite()`

### i18n
```

zh: favoritesTab: "收藏", noFavorites: "还没有收藏歌曲", removeFavorite: "取消收藏"
```

## 2. 品味页面 — 详细用户画像 + 可编辑

### 当前状态
`ProfilePage.tsx` 显示基础统计（总播放、分钟数、收藏数、艺术家排行、年代/语言/心情分布）。
`ProfileService` (profile.service.ts) 有 inferredArtists, inferredGenres, moodHistory。
`/api/profile` 从 plays.repo.ts 获取数据。

### 需求
重塑品味页面，分为以下几个区域：

#### 2a. 音乐画像卡片（只读，自动生成）
- 总播放次数、总收听分钟、收藏数
- 最常听艺术家 TOP 5（带播放次数条形图）
- 收听场景偏好（moodPreference 饼图/条形图）
- 最近收听主题

#### 2b. 我的偏好（可编辑）
用户可以手动编辑以下字段，存储在 ProfileService 的 profile.json 中：
- **喜欢的音乐风格** (favoriteGenres): 字符串数组，如 ["Lo-fi", "Jazz", "Indie Pop"]
- **不喜欢的风格** (dislikedGenres): 字符串数组，如 ["重金属", "说唱"]
- **常听场景** (preferredScenes): 字符串数组，如 ["写代码", "睡前", "通勤"]
- **心情偏好** (preferredMoods): 字符串数组，如 ["放松", "专注", "治愈"]
- **个人备注** (userNote): 自由文本，如 "喜欢日系音乐，不喜欢太吵的"

编辑方式：
- 每个区域显示为标签（tag pills）+ 添加/删除按钮
- 点击标签可以删除
- 有输入框可以添加新标签
- 备注是 textarea
- 有「保存」按钮

#### 2c. 每日推荐历史
- 显示最近 7 天的推荐记录（来自 dailyRecommendations）
- 每天显示推荐的歌曲列表

### 后端改动

#### ProfileService 新增字段
```typescript
export interface UserProfile {
  // ... existing fields ...
  favoriteGenres: string[];      // 用户手动设置
  dislikedGenres: string[];      // 用户手动设置
  preferredScenes: string[];     // 用户手动设置
  preferredMoods: string[];      // 用户手动设置
  userNote: string;              // 用户自由备注
}
```

#### 新增 API
- `PUT /api/profile/preferences` — 更新用户偏好（favoriteGenres, dislikedGenres, preferredScenes, preferredMoods, userNote）
- `GET /api/profile/full` — 返回完整画像（统计数据 + 用户偏好 + 推荐历史）

#### 修改 profile routes
在 `routes/profile.ts` 中添加新端点。

### 前端改动
- `ProfilePage.tsx`: 完全重写
- `api/client.ts`: 添加 `getFullProfile()` 和 `updateProfilePreferences()` API

## 3. 设置页面 — 更多设置项

### 当前状态
`SettingsPage.tsx` 只有 API Key 配置和服务状态。

### 需求
在现有 API 配置下方添加更多设置区域：

#### 3a. 音频设置
- 音频质量：标准 (128kbps) / 高 (320kbps) / 无损 (lossless)
- 存储在 settings 表：`audio_quality`

#### 3b. 频谱可视化
- 开关：启用/禁用频谱动画
- 存储在 settings 表：`spectrum_enabled`

#### 3c. AI 助手设置
- 猫娘回复语言：中文 / English / 日本語
- 每日推荐：开启/关闭
- 存储在 settings 表：`ai_language`, `daily_recommend_enabled`

#### 3d. 关于
- 版本号：1.0.0
- GitHub 链接
- 技术栈说明

### 后端
settings 表已有 key-value 存储，无需改动 schema。
`PUT /api/settings` 已支持批量更新。

### 前端改动
- `SettingsPage.tsx`: 添加新区域
- 新增设置项使用 toggle switch 和 select 组件

## 4. ContextService 集成用户偏好

### 需求
修改 `context.service.ts`，在生成上下文时：
- 如果用户设置了 favoriteGenres，在上下文中注入「用户喜欢的音乐风格」
- 如果用户设置了 dislikedGenres，在上下文中注入「用户不喜欢的风格」
- 如果用户设置了 userNote，在上下文中注入「用户备注」

这样猫娘 AI 推荐时会考虑用户的手动偏好。
